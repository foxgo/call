# Call Task Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a new `call-task` service that owns task management, list import, scheduling, RocketMQ dispatch, and dial-result writeback with MySQL as the source of truth and Redis as the hot scheduling state machine.

**Architecture:** Create a dedicated Spring Boot module `call-task`, move task-domain persistence into `call-common`, keep dial units in MySQL until a running task needs a hot scheduling window, and drive dispatch through Redis `ready/processing/retry` queues plus RocketMQ. Implement task APIs, import-batch tracking, scheduler preload, dispatch claiming, timeout recovery, retry requeue, and writeback idempotency as one closed loop.

**Tech Stack:** Java 21, Spring Boot 3.2, MyBatis-Plus, Spring Data Redis, RocketMQ Spring Boot starter, Micrometer, JUnit 5, Mockito
